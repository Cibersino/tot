toT

How to use

  • Basic guide
  • Full instructions
  • Frequently asked questions (FAQ)

toT is a desktop app that turns text into estimated reading time and helps you plan realistic reading workloads. It combines file text import/extraction from text and image files, configurable WPM presets, precise counting, text snapshots, a task editor, a stopwatch, and a reading speed test so you can measure, organize, and complete readings with less guesswork.

Basic guide

Objective: get familiar with the app.

  1. Import a model page (recommended onboarding flow)

    • Take a good photo of a representative page from any book and save it in any folder on your computer.
    • Choose a base WPM using a preset or the slider/input (a default preset is fine to start).
    • Press 📥 or drag/drop that photo onto the main window.
    • If this is your first time using the app's OCR (Optical Character Recognition) or you don't have it connected to a Google account, follow the steps to authorize and connect the Google OCR service to Google Drive.
    • The text extraction from the image will begin. When it finishes, in the final apply modal choose Overwrite and apply that representative page once.
    • Open the Manual Editor (⌨) to inspect the extracted result. If necessary, make adjustments.
    • Then, in the results section, use the multiplier below the estimated time with the book's total page count to project the full estimate.

  2. Create a custom preset with the reading speed test

    • Once the current text is ready, click Reading speed test.
    • In the entry modal, choose Start with current text.
    • Read that text under normal conditions and use the Floating Window to pause when you finish.
    • When the flow ends, save the suggested preset based on the measured WPM.
    • Warning: very short tests, especially one-page samples, tend to overestimate sustainable speed on longer texts. Use it as an initial calibration, not as a final truth.

  3. Copy from clipboard

    • Use 🗑 to start from scratch.
    • From any source: copy the text and use 📋↺.
    • If you are assembling a text in parts: use 📋+ as many times as you need
    • To repeat the clipboard text in one click, set the number in Repetitions and then apply the action: 📋↺ to overwrite or 📋+ to append.

  4. Edit the text (if needed)

    • Open Manual Editor (⌨).
    • Correct everything weird: line breaks, hyphenation or “garbage” (menus, ads, repeated headers).
    • You can modify or add whatever you want, by typing or pasting in the Manual Editor.

  5. Choose your reading speed (WPM)

    • If you don't want to think too much, select a preset (“Default”, etc.).
    • If the text is dense/technical, lower the WPM; if it is light, turn it up.

  6. Choose the counting mode

    • Precise: recommended in general, and especially if your text has many signs, mixed scripts or emojis (uses linguistic segmentation and graphemes)
    • Simple: rudimentary approach based on spaces

  7. Read the results

    • Check words, characters and the estimated time
    • Use the multiplier below the estimated time when you want to project that same estimate N times without duplicating the current text.
    • If the time “doesn't add up”, go back to (2) and (3), or toggle (4) to compare.

  8. Measure your real WPM (optional, but recommended)

    • With the current text already finished, start the Stopwatch (▶) when you start reading, and pause (⏸) when you finish.
    • Use the real WPM as a calibration to adjust your presets.

  9. If you are going to read in another app

    • Activate the Floating Window to control the Stopwatch without changing windows.

If something goes wrong (rapid diagnosis):

  • “Weird” pasted text (PDF/columns): go through Manual Editor and clean up line breaks/hyphens.
  • Absurd counts: check the text in the Manual Editor (copies from PDF often bring noise).
  • Estimate too high/too low: review your WPM and test Simple vs Precise.

Do you find it hard to finish reading and abandon them in the middle?
Do you want to overcome your difficulties and develop your reading skills using scientific measurement and organization tools?
Does your browser accumulate a lot of tabs and bookmarks with news and articles of interest that you don't know if you will be able to read?
Do you have to teach a class this semester and must provide your students with a realistic bibliography?
Do you have to prepare a paper for tomorrow to present to an audience for one hour?
Do you need to script an audiovisual capsule with precise timing?
Is the pile of books on your bedside table growing mercilessly?
Do you want to do experimental studies related to reading time?

This app is designed for:

  • Import or extract text from txt, md, html, htm, docx, rtf, odt, pdf, and image files (png, jpg, jpeg, webp, bmp, tif, tiff), with OCR for images and scanned PDFs.

Step 1 — Select and assemble the text to be measured

Your first decision is how to construct the text you wish to estimate. From here on we will call it current text.

Route 1A — Import files (recommended primary route)

Use the 📥 button to import or extract text directly from files, with automatic routing by file type. You can also drag and drop a file onto the main window: it runs the same import flow as 📥.

  • Text/document files: .txt, .md, .html, .htm, .docx, .rtf, .odt.
  • Images: .png, .jpg, .jpeg, .webp, .bmp, .tif, .tiff (OCR).
  • PDF with selectable text: choose Native or OCR extraction.
  • Scanned PDF without selectable text: OCR extraction.
  1. Click 📥 and choose the file, or drag/drop the file onto the main window (while dragging files, a visual drop highlight is shown).
  2. If a selectable-text PDF is detected, choose Use native or Use OCR (recommendation: try native first, it's faster).
  3. When text is ready, in the final apply modal choose Overwrite or Append.
  4. The final apply modal also includes Repetitions if you want to repeat the extracted text before applying it.
  5. If you want to approximate a full book from one model page, apply that page once and then use the multiplier below the estimated time with the book's total page count.

Note: this route may ask for Google authorization because some extraction paths are performed through Google, including OCR. The file you choose for that Google-backed extraction is not sent to the developers and is not shared with other people as part of that process. Final quality still depends on source quality (photo/PDF/format).

Recommendations for better OCR results:

  • Prefer clear, sharp photos or scanned PDFs, with the page straight, centered, and vertically aligned.
  • Avoid bent or curved pages, tilt, blur, shadows, and glare.
  • Try to show only the page: avoid parts of other pages, fingers, objects, or cluttered backgrounds.

Route 1B — Use Clipboard (from web/PDF/any source)

  1. Select the text in the source (article, document, chat, etc.). Options:

    • All text in the open document/page. Right-click and “Select all”, or: Ctrl+A (Windows/Linux); Cmd+A (Mac).
    • Only a part of the text (manual selection). For that you can:
      • Left-click at the beginning of the desired text and drag the cursor to the end of the desired text.
      • Select, as above, only a small part of the text at the beginning or at the end of the desired text, move the cursor to the other end of the desired text and Shift + left-click. You can adjust by holding Shift and using the arrow keys.

  2. Copy. Right click and “Copy” or: Ctrl+C (Windows/Linux); Cmd+C (Mac).

  1. In the main app window, choose one of these actions:

    • 📋↺: completely replaces the current text with the clipboard.
    • 📋+: adds the clipboard to the end of the current text.
    • Paste repetitions input (next to 📋↺ and 📋+): sets how many times the clipboard block is applied with a single click.

    Note: with N=2, one click on 📋↺ overwrites once and repeats once with a new-line separation; one click on 📋+ appends twice. If the value is invalid (empty, 0, negative, decimal, or text), 1 is used.

When to use each one:

  • Use 📋↺ when you are going to work with a single text or want to restart.
  • Use 📋+ when you are assembling a text “by parts” (different texts, fragments, quotes, sections, chapters).

Fast quality control:

  • If you copy from PDF or formatted sources/documents, check for extraneous line breaks, dashes, or invisible spaces. If you see problems, go through the Manual Editor (Path 1C or Step 2).

Route 1C — Type or paste directly into the Manual Editor

  1. Open Manual Editor with the button ⌨.
  2. Paste or type the text directly there.
  3. You can undo/redo as in any text editor: Ctrl+Z / Ctrl+Y (Windows/Linux); Cmd+Z / Cmd+Shift+Z (Mac).
  4. You can also perform searches: Ctrl+F (Windows/Linux); Cmd+F (Mac).

Note: The Manual Editor, in addition to updating and saving the current text immediately, by default recalculates the count and generates the estimate automatically. If you want the latter to be manual, you can remove the “calculate while typing” check in the editor window and use the CALCULATE button.

When this route is convenient:

  • You need to clean up formatting (PDF, columns, dashes).
  • You will be doing a lot of editing (correcting, trimming, reorganizing).
  • You want total control over the final document.

Route 1D — Mixing (recommended for real texts)

  1. Load a quick draft with 📋↺.
  2. Use 📋+ to add missing sections.
  3. Open Manual Editor to:
    • remove garbage (repeated headers, numbering, line breaks),
    • correct paragraphs,
    • delete content that you do not want to include in the measurement,
    • leave a “clean” final version.

Note: You can completely empty the current text quickly with the 🗑 button in the main window or in the manual editor.

Route 1E — Save / Load snapshots

If you frequently reuse texts, you can save and load snapshots of the current text using the 💾 and 📂 buttons.

  • 💾: first opens an in-app modal where you can optionally tag the snapshot by language, type, and difficulty. After you confirm there, the app opens a native “save file” dialog rooted at config/saved_current_texts/. You can save into subfolders (and create folders) as long as you stay inside that snapshots tree. The snapshot is saved as JSON.
  • 📂: opens a native “open file” dialog and lets you pick any snapshot file inside config/saved_current_texts/ (including subfolders). The app will ask for confirmation because it will overwrite the current text.
  • A loaded snapshot becomes the new current text (same semantics as other overwrite actions) and the preview/results update immediately. Saved snapshot tags remain in the snapshot file; they are not transferred into the current-text state.

Step 2 — Verify the current text (checkpoint)

Before calculating or measuring, make a minimum verification:

  • Is the text complete?
  • Does it include material you don't want to measure (repeated titles, web navigation, ads)?
  • Are there copying artifacts (many line breaks, dashes “-” at the end of each line)?

If the answer is “yes”, go back to the Manual Editor and correct. This step usually avoids errors in real WPM and estimate.

Step 3 — Adjust the reading speed (WPM) for the estimation

Here you decide at what speed the time will be estimated.

3.1 Choosing a WPM

You have three options:

  • Adjust the WPM manually. You can:
    • Type directly in the input.
    • Move the point on the horizontal bar (slider).
  • Choose a preset from the preset dropdown list. Here you can have the following options:
    • Default presets for all languages and/or for specific languages.
    • User-customized presets.
  • Create a custom preset from the reading speed test (see Step 6), especially if you already have a representative current text and want a guided calibration.

Operational recommendation:

  • If your goal is “a reasonable estimate”, use a “normal reading” preset.
  • If the text is technical/dense, use a slower preset.
  • If it is light reading, you can use a faster one.

3.2 Manage presets (if needed)

  • Create a preset if you are missing one for your typical case (technical, screen, language, etc.).
  • Custom presets can be created manually, for example by using a Stopwatch-based measurement (see Step 8), or in a more automated way through the reading speed test (see Step 6).
  • Edit if your preset is systematically high/low or you want to edit its name or description. You can also edit the default presets.
  • Delete if it became noisy.
  • Restore default presets if you want to return to the original set of default presets that were edited or “deleted” (does not remove “pure” custom presets).

Step 4 — Decide on counting mode (simple vs. precise)

4.1 Differences between the two modes

Simple mode:

  • Words: separated by spaces (and equivalent spaces).
  • Characters: counts characters directly on the text (with and without spaces).

Precise mode:

  • Characters: segments by graphemes (closest to “perceived characters” by the user, useful for emojis and combinations).
  • Words: segments with linguistic rules when available in the system (per language), and avoids counting purely punctuation tokens.

What changes in practice:
They change the counts (words/characters) and therefore change the estimated time and the real WPM.

4.2 Technical explanation

  • Simple mode: calculates words by separating the text by spaces (whitespace). For characters, it counts units of the string (and then distinguishes with/without spaces).
  • Precise mode: uses system Intl.Segmenter to segment:
    • Graphemes to count perceived characters.
    • Words to count words, using the signal isWordLike when it exists (and an equivalent heuristic when it does not), to avoid counting tokens that are just punctuation.
    In addition, the app applies a binding rule (alnum join): patterns like alnum(-alnum)+ are counted as one word (e.g. e-mail).

4.3 Practical example (text + what to observe)

Paste this text as “current text” and toggle between simple and precise: 

alpha / beta / gamma
我喜欢读书因为它很有趣
Emojis: 👨‍👩‍👧‍👦 + 📚 + 👍🏽

Remember that Repetitions duplicates text in the current text, either from the main selector (📋↺/📋+) or from the import/OCR final apply modal. Use the multiplier below the estimated time when you want to project time without expanding the current text.

What to see:

  • In alpha / beta / gamma, the simple mode inflates “words” by including punctuation tokens isolated by spaces (e.g. /).
  • In languages without spaces between words (e.g. Chinese), simple mode counts all pasted characters as a single "word", while precise mode segments them into more units.
  • In the emoji line, compare the character count: precise mode aligns better with what a user perceives as “a character”.

Step 5 — Read results (counts and estimated time)

With current text + WPM + counting mode:

  • Check words, characters with/without spaces.
  • Check estimated time.
  • Use the multiplier below the estimated time if you want to project that same estimate N times without duplicating the current text (for example, one representative page times a total page count).

If estimated time “does not make sense”:

  1. check that WPM is not absurd,
  2. check text (Step 2),
  3. compare simple vs. precise (Step 4).

Step 6 — Reading speed test

This flow is designed to calibrate a custom WPM through a guided measurement and turn that result into a reusable preset.

6.1 When it is useful

  • When you want to move from a rough estimate to an empirical calibration.
  • When you already have a representative current text and want to measure your pace on that exact material.
  • When you want to build more realistic presets for different types of reading.

6.2 “Current text” path

  1. Leave the current text exactly as the sample you want to use.
  2. Click Reading speed test.
  3. In the entry modal, choose Start with current text.
  4. The Editor and the Floating Window will open; read normally.
  5. When you finish, pause pressing (⏸) in the Floating Window.
  6. The app will compute the WPM and open the preset modal so you can save it as a custom preset.

Note: you can cancel the test pressing (⏹) in the Floating Window. If you cancel this path, the current text is preserved.

6.3 “Random text” path

  1. Click Reading speed test.
  2. In the entry modal you can filter by language, type, and difficulty.
  3. Press Start with random text so the app chooses an eligible text from the pool.
  4. Read that text in the guided session and pause from the Floating Window when you finish.
  5. If that text includes comprehension questions, they will appear before the preset modal.

Note: you can cancel the test pressing (⏹) in the Floating Window. If you cancel this path, the current text is not preserved.

6.4 Interpretation and warnings

  • Use the result as empirical calibration, not as immutable truth.
  • Very short tests, especially one-page samples, tend to overestimate sustainable speed on longer texts.
  • If you want a more representative measurement for long readings, use a longer sample with difficulty similar to what you normally read.

Step 7 — Tasks: plan and track readings

The Tasks feature is optional. It helps you manage reading plans as lists of texts with estimated time, progress, and links.

7.1 Open the Task Editor

  • From the main window, open the Task Editor using the Tasks controls: 📝 (new task) or 🗃️ (load task).
  • 📝 opens an empty task list.
  • 🗃️ lets you pick an existing saved task list.

7.2 Add and edit items (rows)

Open de task editor with 📝. Yo can add rows clicking “Add row”. Each row represents one reading item with these columns: Text, Time, % (percent complete), R (remaining time), Type, Link, Comment, and Actions.

  • Enter the item title/description in Text (e.g. “Las mil noches y una noche”).
  • Set an estimate in Time (for example, paste the estimate you got in Step 5).
  • Update % with your approximately percentage completed to track progress; the remaining time in R updates accordingly.
  • Use Type to categorize items as you like (e.g. “url”, “pdf”, “book”, “paper”).
  • Use Link to store a local path or an HTTPS URL for the item.
  • Use 💬 (comment) for any notes you want to add about the item and/or link it to any text snapshot you have saved (a clickeable icon 📥 will appear).

7.3 Links

The Link column can be used to store a local path or an HTTPS URL for the item.

  • For local paths, paste the absolute path of the file (e.g. C:\Users\user\Documents\mybook.txt).
  • For URLs, make sure to include the protocol (e.g. https://).
  • Clicking ↗️ will attempt to open it with the default application (e.g. a PDF will open in the default PDF viewer, a URL will open in the default web browser).
  • When you click on a URL link, the app may ask for confirmation (and may remember trusted hosts) before opening it in the default browser.
    • >

Note: The trusted hosts are remembered in a list that is persisted in the user configuration folder.

7.4 Save, load and delete tasks

  • To save a task list choose a name for your task and click “Save” in the Task editor window. It will open the save folder, where you must choose a file name (a JSON file) and click "Save".
  • For your safety, there is no auto-save. If you want to update a saved task, you must save it again. You can overwrite the file in the folder if you don't want to create another task.
  • To load a previously saved task list, click the 🗃️ (load) button in the Main window.
  • To delete a task, load it and click the “Delete” in the Task Editor.

Note: Saved tasks are stored in the user configuration folder under config/tasks/lists/. You can manage them directly from the file system if needed.

7.5 Library (reusable rows)

In the Tasks editor window you can save a row into a reusable Library. This is useful for items you read frequently or use in different tasks. To save a row to the Library, click the 💾 (save to library) action in that row. The saved entry will be available in the list of the Library, clicking Load library in the Task Editor. From there you can load in your task any row you saved.

7.6 Column widths

You can resize table columns; widths are persisted for the Task Editor.

Step 8 — Measure your real WPM with the Stopwatch

This step is for calibration or actual measurement.

8.1 Preparation (what usually fails)

  • Make sure that the current text is exactly what you will read.
  • Do not edit the text during the measurement.
  • Decide whether you will read the whole text or just a part of it (and adjust the current text to that part).

8.2 Measurement

  1. Start (▶) the Stopwatch as you begin reading.
  2. Pause (⏸) at the end. Attention: do not press Stop (⏹) unless you want to reset the Stopwatch and return to the beginning.
  3. Observe the real WPM calculated.

Interpretation:

  • If the real WPM is consistently lower than your preset, lower that preset.
  • If it is consistently higher, you can raise it.

Notes:

  • You can manually insert an arbitrary time into the Stopwatch. Note: this will remove accumulated (invisible) fractions of a second, if any.
  • If the estimated time doesn't exactly match your stopwatch time for the same apparent WPM, it's probably due to the difference between fractions of WPM not displayed on the stopwatch and the WPM at the selected speed. Additionally, but to a lesser extent, it could be due to fractions of a second not displayed in the estimate and/or the stopwatch, and/or rounding.

8.3 Floating Window (if you read in another app)

Use the Floating Window to control the Stopwatch without constantly switching windows.

Step 9 — Local saving, cleaning and “clean” reinstallation

9.1 What is stored locally

The app saves data locally to maintain your settings between sessions. In general it includes:

  • Current text (text status).
  • Saved text snapshots (optional library under config/saved_current_texts/).
  • Configuration (e.g. language, current WPM, counting mode).
  • Google OCR setup state: app-managed local Google OAuth credentials and, when OCR is connected, the local Google OCR token state.
  • Presets (custom and default preset status).
  • Reading speed test: local test-text pool and its external local usage state file.
  • Status of windows (e.g. size/position of the editor).
  • Tasks: task lists, task library, link allowlist, and Task Editor UI state (position and column widths).

Implication:

  • If you work with sensitive text, use 🗑 at the end and consider total cleanup.

If you only want to disconnect Google OCR without deleting the rest of the app configuration, use Menu > Preferences > Disconnect Google OCR. On success, the app revokes the saved Google token and deletes the local OCR token file. App-managed local Google OAuth credentials may remain so OCR can reconnect later. As an external fallback/control, you can also revoke the app's access from your Google Account permissions/security controls.

9.2 Total cleanup (clean reinstallation) — with paths per system

To return to “first run” status:

  1. Close the app (this avoids half-written files).
  2. Delete the local configuration folder (config).

Typical paths (@cibersino/tot app folder):

  • Windows: %APPDATA%\@cibersino\tot\config\ (equivalent to C:\Users\<YOUR_USER>\AppData\Roaming\@cibersino\tot\config\)
  • Linux: ~/.config/@cibersino/tot/config/ (or $XDG_CONFIG_HOME/@cibersino/tot/config/)
  • macOS: ~/Library/Application Support/@cibersino/tot/config/
  1. Open the app and reconfigure (language, presets, etc.).

Step 10 — Limits and very long texts

  • For security and performance, the app imposes a hard limit on the current text size of 50 million characters. If you try to exceed it, the app truncates the text and displays a warning.
  • With extremely long texts (or complex combinations of characters) there may be delays in counting and increased memory usage.

Note: 50 million characters is roughly equivalent to about 5 times the 7 volumes of Marcel Proust's In Search of Lost Time (about 500 hours of reading for a median adult).

Frequently asked questions (FAQ)

“Are there apps similar to toT?”

Yes, there are several related tools:

  • Web calculators: estimate time from a pasted text.
  • Browser extensions: show “estimated time” in pages.
  • Reading/annotation apps: metrics, progress, counting or markers.

What is distinctive about toT is that it works as an operational tool: build a current text (clipboard and/or editor), set WPM/presets, compare Simple vs Precise, measure real WPM with Stopwatch (including Floating Window), and use a guided reading speed test to calibrate presets from either the current text or pool texts.

“How is the development and maintenance of toT financed?”

toT is a project maintained by its developers as general purpose software.
It does not depend on monetizing your text or “selling data”.
No advertising.

If you want to support its development, you will be able to do it through a donation/support link when it is published in the repository.

You can also contribute:

  • Reporting problems with clear reproduction steps.
  • Proposing concrete improvements (UX, performance, i18n, documentation).
  • Contributing translations or small and verifiable PRs.

“Why is WPM used to estimate reading time?”

Because WPM (words per minute) is a simple, standard and reasonably stable parameter to make a first approximation.

Unavoidable limitations:

  • The real WPM changes according to difficulty, language, reading mode (silent or read aloud, etc), and objective (deep comprehension vs. speed reading).
  • That is why WPM is most useful when you treat it as a tunable parameter, not as “truth”.

The app includes a Stopwatch precisely so you can measure your real speed and adjust presets empirically. In addition, the reading speed test provides a more guided route for creating custom presets, especially useful when you already have a representative current text.

“How does the “precise mode” work?”

On a technical level, the Precise mode seeks to segment the text in a more linguistic way:

  • Use Intl.Segmenter to segment Words.
  • Use Intl.Segmenter to segment Graphemes by counting "perceived" characters (useful in emojis and combinations).
  • Avoid counting tokens that are only punctuation.
  • Applies a join rule (alnum join): alnum(-alnum)+ patterns are counted as one word (e.g. e-mail).

The Simple mode is a space-based approach: it separates “words” by whitespace and counts characters directly on the string.

“Does the app protect my privacy?”

Essentially, toT operates locally. The text you paste or type is processed on your machine to count/estimate.

Things to distinguish:

  • Local storage: the app saves data locally (for persistence between sessions). This includes:
    • Current text and general configuration (language, WPM, counting mode, presets).
    • Saved text snapshots (optional library under config/saved_current_texts/).
    • Reading speed test: local test-text pool and its external local usage state file.
    • Google extraction setup state: app-managed local Google OAuth credentials and, when the Google-backed extraction path is connected, the local Google token state.
    • Saved tasks (task lists, task library and link allowlist).
  • Internet: the app may require connection when you execute actions that use external resources, such as:
    • open external links clicked by the user (including links opened from info pages and links opened from Tasks),
    • check/open update information (e.g. on GitHub),
    • if you choose to use a Google-backed extraction path: authorize Google in the system browser and send the files you explicitly selected for that processing.

Practical implication: ordinary pasted/typed text handling in toT remains local. However, actions that open an external site expose to that service your IP address and normal connection metadata (as in any browser). In addition, if you choose to use a Google-backed extraction path, the files you explicitly select for that processing are sent to Google for the operation; they are not sent to the developers or to other third parties as part of that flow. The app also attempts immediate remote cleanup of the temporary converted Google document after export. If you want to disconnect Google OCR only, use Menu > Preferences > Disconnect Google OCR. On success, the app revokes the saved Google token and deletes the local OCR token file. App-managed local Google OAuth credentials may remain so OCR can reconnect later. If you also want to remove the app's access from Google's side, revoke it from your Google Account permissions/security controls. If you want to remove all locally stored data, see Step 9 (“Local saving, cleaning and “clean” reinstallation”).

“What happens if I enter a very long text?”

The app has a hard limit for the current text size (for performance and security): 50 million characters. If you exceed it, the text is truncated and a warning is displayed.

To give you an idea: 50 million characters is roughly equivalent to about 5 times the 7 volumes of the famous “In Search of Lost Time” by Marcel Proust. This is equivalent to about 500 hours of reading for a median adult.

Practical recommendations:

  • Divide giant texts into parts (chapters/sections) and measure separately.
  • If you are building a corpus, use 📋+ and control the size.
  • If copying from PDF/columns, first clean up in the Manual Editor (reduce garbage before growing text).

“Why is the app called toT?”

It is a short name designed to be associated with the central idea of the app: time + text (measurement of reading time from a text).

In addition, it is inspired by the Platonic myth about the Egyptian god Theuth. Plato, in his dialogue “Phaedrus”, tells how this divinity, creator of hieroglyphics and measurer of time, presents to King Ammon the invention of writing as a memory drug. The king replies that it is not a remedy, but simply a reminder.

“How many words does a page normally have?”

It depends very much on the type of page, the format (size, margins, line spacing, typography) and the language/writing system.

If it is a website, it is extremely relative: there is no standard. In addition, many pages include menus, banners, footers, columns, etc., which change the actual “density” of text.

If you are referring to a book or document page, the format is important, but the language is also important (a text in English/Spanish is not the same as one in Japanese, for example).

  • Books in languages with words separated by spaces (Latin alphabet and related): a “standard” book usually contains approximately 250-300 words per page.
  • Word document (letter size, Arial 12, single spacing and standard margins): usually contains approximately 500-600 words per page.

If you need a reliable figure for your case, the most direct way is to take a representative page of the same material (same format and language) and measure it with toT.
